HTTPж–°method: Query

About: IETFжңҖиҝ‘зҡ„дёҖдёӘе…ідәҺHTTPж–°methodзҡ„proposal

GETдёәд»Җд№ҲдёҚеә”иҜҘжңүbody

ж №жҚ® HTTP Spec

GET requestзҡ„payloadжҳҜжІЎжңүе®ҡд№үзҡ„пјҢеҸҜд»ҘзҗҶи§ЈдёәCиҜӯиЁҖдёӯзҡ„undefined behaviorпјҢж–Үз« дёӯиҝҳжҸҗеҲ°дәҶпјҢ
GET bodiesд»ҺдёҚиў« Fetch ж”ҜжҢҒпјҢеҸҳжҲҗдәҶиҷҪз„¶ж”ҜжҢҒпјҢдҪҶжҳҜдјҡиӯҰе‘ҠејҖеҸ‘иҖ…е°ҪйҮҸдёҚиҰҒз”ЁгҖӮ

еӣ дёәжҳҜ undefined behaviorпјҢжүҖд»Ҙ GET иҜ·жұӮдёӯзҡ„ BodyпјҢзҗҶи®әдёҠжқҘи®ІеҸҜд»Ҙиў«йҡҸж„ҸеӨ„зҗҶпјҢеғҸHackerNews
дёҠиҝҷдёӘ Comment жүҖиҜҙзҡ„:

GETзҡ„bodyжҳҜе…·жңүдёҚзЎ®е®ҡжҖ§зҡ„пјҢLoad BalancerзҗҶи®әдёҠжҳҜеҸҜд»ҘзӣҙжҺҘдёўжҺүBodyеҶ…е®№зҡ„гҖӮ

ж–°Method вҖңQUERYвҖқжңүд»Җд№ҲеҘҪеӨ„

Safe and Idempotent

Proposal Section 2
йҮҢжҸҗеҲ°зҡ„жңҖйҮҚиҰҒзҡ„еҘҪеӨ„жҳҜsafeе’ҢIdempotentгҖӮ

еҜ№жҜ”POSTпјҢQUERYдёҚеә”ж”№еҸҳжңҚеҠЎеҷЁдёҠзҡ„еҶ…е®№пјҢд№ҹе°ұжҳҜеҸӘиҜ»иҜ·жұӮпјҢиҖҢдё”QUERYеҗҢж ·д№ҹеә”иҜҘжҳҜе№Ӯзӯүзҡ„пјҢ
вҖңд»ҘзӣёеҗҢзҡ„иҜ·жұӮи°ғз”ЁиҝҷдёӘжҺҘеҸЈдёҖж¬Ўе’Ңи°ғз”ЁиҝҷдёӘжҺҘеҸЈеӨҡж¬ЎпјҢеҜ№зі»з»ҹдә§з”ҹзҡ„еҪұе“ҚжҳҜзӣёеҗҢзҡ„вҖқгҖӮ

Caching

Proposal Section 2.1 жүҖжҸҗеҲ°зҡ„пјҢQUERYж”ҜжҢҒHTTP cachingзҡ„ж ҮеҮҶгҖӮ

з»јеҗҲжқҘзңӢпјҢQUERYзӣёеҪ“дәҺжҳҜдёҖдёӘж”ҜжҢҒдәҶBodyзҡ„GETгҖӮеҜ№жҜ”дј з»ҹйҖҡиҝҮPOSTжқҘдј Bodyе®һзҺ°еӨҚжқӮжҹҘиҜўпјҢжңүдёҖдәӣдјҳеҠҝгҖӮ

Thoughts

HTTPе®ҡд№үзҡ„иҝҷдәӣж–№жі•еңЁеҪ“еүҚзңҹзҡ„з¬ҰеҗҲеҗ„дёӘдёҚеҗҢеә”з”Ёзҡ„йңҖжұӮеҗ—пјҢHTTPжҳҜеҗҰеә”иҜҘеңЁе®ғиҝҷдёҖеұӮеҺ»е®ҡд№үиҝҷдәӣж–№жі•пјҢ
жҳҜеҗҰеә”иҜҘз”ұдёҖеұӮеҸӘеӨ„зҗҶеҚ•еҗ‘жҲ–еҸҢеҗ‘зҡ„ж¶ҲжҒҜйҖҡи®ҜеҚҸи®®пјҢ е…¶дҪҷзҡ„йҖ»иҫ‘пјҢз”ұжӣҙдёҠеұӮзҡ„еҚҸи®®еҺ»еҒҡгҖӮ
еғҸGraphQLеңЁHTTP POSTд№ӢдёҠе®ҡд№үиҮӘе·ұзҡ„Queryе’ҢMutationйҖ»иҫ‘гҖӮ

Kotlin Coroutine LockFreeTaskQueue жәҗз ҒжіЁйҮҠ

Lock Free Task Queue

еҜ№ Kotlin Coroutine зҡ„Lock Free Task Queueзҡ„жіЁйҮҠ

еёёйҮҸ

InternalеҸҳйҮҸ

зҠ¶жҖҒж–№жі•

Placeholderзұ»пјҢеё®еҠ©ж Үи®°зҠ¶жҖҒзҡ„дёҖдёӘеҶ…йғЁзұ»

addLast - producerж·»еҠ д»»еҠЎ

removeFistOrNull consumerжқҘжӢҝд»»еҠЎпјҢеҰӮжһңжІЎжңүд»»еҠЎпјҢеҲҷиҝ”еӣһnull

жү©е®№йҖ»иҫ‘пјҢеңЁaddLastзҡ„ж—¶еҖҷпјҢж»Ўи¶ід№ӢеүҚжҸҗеҲ°зҡ„жқЎд»¶дјҡиҝӣиЎҢжү©е®№

жү©е®№е®һйҷ…жҳҜcopyд№ӢеүҚзҡ„arrayйҮҢиҫ№зҡ„е…ғзҙ еҲ°ж–°зҡ„arrayйҮҢиҫ№

жңҖеҗҺLockFreeTaskQueueпјҢиҝҷйҮҢиҫ№зҡ„йҖ»иҫ‘жҜ”иҫғstraight forward

жҖ»з»“

еҹәжң¬жҜҸдёҖдёӘиҰҒеӨ„зҗҶејӮжӯҘзј–зЁӢзҡ„libraryйғҪйңҖиҰҒжңүдёӘqueueжқҘз»ҙжҠӨд»»еҠЎпјҢKotlinиҝҷдёӘcoroutineеӣ дёәиҰҒtarget multiplatformпјҢ
еҶҷиҝҷд№ҲдёӘqueueиҰҒиҖғиҷ‘еҫҲеӨҡcaseпјҢеӣ дёәжңүзҡ„platformеҸҜиғҪжҳҜsingle consumerпјҢжңүзҡ„еҸҲжҳҜmulti consumerпјҢ
дёәдәҶиҝҷдёӘиҝҳеҶҷдәҶдёҚеҗҢзҡ„еӨ„зҗҶйҖ»иҫ‘пјҢд№ҹз®—жҳҜеҒҡеҲ°жһҒиҮҙдәҶгҖӮ

еҜ№жҜ”еҲ«зҡ„libraryеҸҜиғҪеҖјйңҖиҰҒеҒҡеҲ°MPSCпјҢжҜ”еҰӮReactorзҡ„ MpscLinkedQueue
пјҢkotlin coroutineзҡ„queueзҡ„йҖ»иҫ‘иҰҒеӨҚжқӮеҫ—еӨҡгҖӮ

еңЁж•ҲзҺҮж–№йқўпјҢеҰӮжһңжҳҜеҜ№дәҺtargetжҳҜJVMзҡ„пјҢеә”иҜҘи·ҹз”ЁзәҝзЁӢжұ е®һзҺ°NIOзҡ„ж•ҲзҺҮе·®дёҚеӨҡпјҢжҲ‘и®°еҫ—д№ӢеүҚзңӢжү”зү©зәҝзҡ„ Kotlinзҡ„еҚҸзЁӢ
жҸҗеҲ°иҝҮиҝҷдёӘжҰӮеҝөпјҢKotlinзҡ„coroutineе№¶дёҚдјҡжҜ”зәҝзЁӢжұ жӣҙж•ҲзҺҮпјҢеӣ дёәе®ғеә•еұӮд№ҹжҳҜиҝҷд№Ҳе®һзҺ°зҡ„гҖӮ

иҖҢдё”д»ҺеҸҰдёҖдёӘи§’еәҰзңӢпјҢеңЁJSйҮҢпјҢиҝҷз§ҚеңЁKotlinз»ҙжҠӨдёҖдёӘtask queueпјҢжңӘеҝ…жҜ”nativeзҡ„event loopжӣҙж•ҲзҺҮпјҢ
еӣ дёәKotlinжңҖз»ҲжҳҜtranspileжҲҗдәҶJSпјҢиҖҢNodeйҮҢиҫ№зҡ„event loopжҳҜlibuvз”ЁCе®һзҺ°зҡ„гҖӮ

дёҚзҹҘйҒ“иҝҷдёӘжғіжі•еҜ№дёҚеҜ№гҖӮ
